# 05 观测与告警模块 开发任务清单

## M1 埋点
1. 调度/队列/run/step/tool/SSE/sandbox 指标埋点
2. 结构化日志字段统一
3. trace_id/span_id 贯通
4. SSE 指标拆分：全局摘要流/会话详情流分别埋点

## M2 告警
1. P1/P2/P3 规则落地
2. 去重、抖动抑制、恢复通知
3. 告警事件审计

## M3 看板
1. 平台总览
2. 租户视图
3. Agent 视图
4. 会话诊断联查页

## M4 验证
1. 故障注入验证阈值
2. 跨租户观测隔离验证
3. 在线自动触发场景下，摘要流推送延迟与错误率达标

## M5 Sandbox+Agent+Gateway 风险告警落地（新增）
1. 将 R-01 ~ R-07 映射为独立告警规则（规则名与 05 主文档一致）
2. 为每条规则配置 `severity/module/owner/tenant_id` 标签
3. Alertmanager 路由：
- `P1 -> 主责值班组 + 备用值班组 + 平台负责人`
- `P2 -> 主责值班组`
- `P3 -> 工作时段队列`
4. 告警模板中固化首个应急动作（Runbook first action）
5. 演练以下场景：
- runner 违规出网（R-01）
- token 吊销后仍可调用（R-02）
- gateway 5xx/高延迟（R-03）
- runner 心跳丢失（R-05）
- 密钥泄漏检测命中（R-06）
6. 输出演练报告：
- 触发时间、通知延迟、响应人、处置耗时、恢复时间、RCA 链接
